# Contributing to Sale Tracker

Thank you for your interest in improving Sale Tracker.

## Before you begin

- Search existing issues before opening a new one.
- Keep proposals focused on a user problem or a measurable improvement.
- For significant changes, open an issue first so the approach can be discussed.

## Pull requests

1. Create a focused branch from the default branch.
2. Make a small, tested change with clear commit messages.
3. Describe the user-facing impact in the pull request.
4. Do not include credentials, personal customer data, or generated build artifacts.

By contributing, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
